# AI 对话应用 产品文档

## 1. 产品定位

一款参考 DeepSeek 风格的网页版 AI 对话产品。支持游客免登录试用与注册用户长期使用两种路径，核心是流式（SSE）AI 问答 + 历史会话管理。

## 2. 目标用户

- 想快速体验 AI 对话、不想注册的访客
- 需要长期保存对话历史、多会话管理的注册用户

## 3. 核心功能

### 3.1 游客试用
- 未登录可直接进入聊天界面提问
- 按 `anonymous_id`（前端生成并持久化）或客户端 IP 做每日次数限额，默认 5 次/天（`GUEST_DAILY_LIMIT` 可配置）
- 超出限额返回 429，前端弹出引导登录的限额提示（`GuestLimitDialog`）
- 游客对话不落库，不保留历史

### 3.2 账号体系
- 注册 / 登录（用户名+密码，JWT 鉴权）
- 查看当前用户信息、修改密码
- 登录后不受次数限制

### 3.3 会话与消息
- 登录用户可创建、重命名、删除会话（`chat_sessions`）
- 每个会话保存完整消息记录（`chat_messages`），支持查看历史
- 侧边栏按时间分组展示会话列表（`ChatSidebar` + `groupSessionsByDate`）

### 3.4 对话能力
- 通过 `/api/chat/stream` 以 SSE 流式返回 AI 回复，前端逐字渲染
- 后端经 langchain（OpenAI 兼容协议）调用第三方 LLM（默认 `deepseek-chat`，即 DeepSeek-V4，支持工具调用；模型/Base URL/Key 均可配置）
- 思考模式下展示模型思维链：AI 气泡上方有可折叠的「💭 思考过程」区块（`MessageBubble`），思维链仅用于展示，不落库、不回灌进后续上下文（历史消息只保留最终回复）
- **工具调用**：模型可调用内置「获取当前日期」工具，以及（可选开启的）MySQL MCP 数据库查询工具
- **回复资源转存**：登录用户回复中内联的图片/视频/文件（base64）落库前自动转存阿里云 OSS、替换为外链
- 支持代码块高亮与一键复制（`CodeBlock` + `CopyButton`）

### 3.5 知识库与 RAG（需登录）
- 知识库管理页（`/knowledge-base`）：上传文档（.txt/.md/.pdf/.docx/.xlsx，含表格与图片）、列出、删除
- 文档区分「全局共享」与「个人」两种可见范围；删除仅限上传者本人
- 导入时自动切分、向量化（Qwen3-Embedding）并存入 Milvus；表格转 markdown、图片转存 OSS 并 OCR 识别文字一并入库
- 对话时 RAG 以「工具」形式接入：模型在需要时自行调用知识库检索（function calling），按所选范围（全局/个人/两者/关闭）召回相关内容来回答
- 检索采用「先召回（Milvus 向量检索）再重排（rerank 交叉编码器）」两阶段，提升进入回答的片段相关性（rerank 可配置开关）
- 登录用户的历史对话也会自动索引进向量库

### 3.6 设置
- 个人设置页（`/settings`），含修改密码等账号管理操作

## 4. 主要页面

| 路径 | 说明 |
| --- | --- |
| `/` | 首页 |
| `/login`、`/register` | 登录、注册 |
| `/chat` | 主对话界面（侧边栏 + 消息列表 + 输入框，登录用户可选知识库检索范围） |
| `/knowledge-base` | 知识库管理（上传/列出/删除文档，需登录） |
| `/settings` | 账号设置 |

## 5. 技术架构

- **前端**：Next.js（App Router）+ TypeScript，Zustand 做状态管理（`useAuthStore` / `useChatStore` / `useSessionStore`）
- **后端**：FastAPI，分层为 `routers`（参数解析与转发）→ `services`（业务逻辑：鉴权、会话归属校验、游客限额、知识库、RAG）→ `models`（SQLAlchemy ORM），`routers` 不直接操作数据库；langchain 细节收敛在 `app/llm` 组件
- **数据库**：MySQL（SQLAlchemy 2.x + PyMySQL）
- **鉴权**：JWT（python-jose）+ bcrypt 密码哈希（passlib）
- **LLM 接入**：langchain-openai（`ChatOpenAI`）走 OpenAI 兼容协议，支持流式与工具调用
- **向量检索**：Qwen3-Embedding 向量化 + Milvus 向量库（知识库与对话索引各用独立集合）+ rerank 重排（DashScope gte-rerank，可选）
- **文档解析**：PyMuPDF / python-docx / openpyxl 解析 PDF/Word/Excel，RapidOCR 识别图片文字，阿里云 OSS 存储图片/资源

## 6. 关键业务规则

- 游客身份识别优先用 `anonymous_id`，IP 仅作兜底，避免误伤同网络下的多个用户
- 会话归属校验在 `session_service` 中统一处理，杜绝跨用户访问他人会话
- 知识库可见性：列出 = 全部全局 + 自己的个人；删除仅限上传者本人（无管理员角色，规则力求简单安全）
- 知识库/RAG/资源转存均为「增强能力」：相关服务（Milvus/Embedding/OSS/OCR/rerank）未配置或失败时自动降级，不影响基础对话
- 开发环境启动时自动建表，生产环境建议改用 Alembic 迁移（当前未接入）
